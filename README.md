### SPO2 Monitor设计

本项目预期基于ZYNQ MPSoC Ultrascale 实现一个血氧仪，目前需要进行软件代码(PS)的设计工作；

### 已经存在，且你应当参考、应用的模块

```

platform/ - PL端设计的所有细节都在其中了，设计时要有嵌入式思维；

/SpO2_app/src

dynclk/ - 为视频流提供时钟的IP Core 驱动
PWM/ - 控制屏幕亮度的PWM IP驱动
display_ctrl/ - 控制视频流的驱动代码，控制VDMA的逻辑应当也在其中；
third_party/ - LVGL UI框架，本项目采用的UI方案；
i2c/ - 屏幕的触摸功能是PS端i2c接口实现的；
touch/ - 屏幕触摸功能相关驱动代码；

```
### 需要实现的模块
```
/SpO2_app/src

FIR/ - 如果 FIR Compiler需要驱动代码，请务必按照Xilinx IP 官方驱动进行移植；
DMA/ - 如果 DMA 需要驱动代码，请务必按照Xilinx IP 官方驱动进行移植；
IIC_PL/ - PL涉及了一个IIC控制器（软核），需要驱动代码，请务必按照Xilinx IP 官方驱动进行移植；
SPO2/ - 和血氧的计算与显示相关的功能模块封装在其中；
HR_algor/ - 根据PPG测算心率的代码；
MAX30102/ - MAX30102的驱动移植代码；务必参考开源代码进行移植工作；
interrupt/ - 你所要处理的中断相关问题的所有代码都放在此处（可选，但如有必要一定要遵守这个规范）；

参考Xilinx官方的推荐与范例，建立一个FreeRTOS任务项目来处理我的所有设计需求，合理划分任务并设定优先级事宜。

如果有其他需要，你也可以自行添加模块，重点是实现需求，在设计时优先参照经过验证的官方代码；


```

### 我的设计需求

从MAX30102中以最大采样率读出红外，远红外的数据，由FIR进行双路滤波，并由DMA处理搬运；通过视频流显示红外、远红外双路波形，并
计算SPO2值，通过类似于监护仪大数字的模式进行显示；心率也要设计算法计算，并通过大数字进行显示；UI由LVGL实现，关键在于波形显示和数字显示这几个要素，一般波形在左，数字在右；波形和监护仪一样，不是滚动刷新，而是从左到右循环刷新。波形的显示应以易读为原则，动态地处理数值范围，确保波形显示是PPG典型、理想的效果。

着重注意MAX30102的驱动读出数据位数和滤波器输入位数、滤波器输出位数间的变化或不同，合理地填0或截断(PL端中可以查到相关的设计)。

此外也进行人机交互的一些设计；例如告警、数字闪烁、变色等这些方式（或是未检测到信号），都可通过UI实现。由于我目前还没有传感器的转接板子，又要测试，因此，我希望这个程序带一些自检功能，是否检测到传感器MAX30102。触摸屏相关功能设计可以之后再说。UI风格优先使用较细的字体，展现科技感，使用英文（中文支持较难）；一个典型的想法，在传感器离线时，波形淡化，上覆盖较大的SENSOR OFFLINE字样；
未检测到信号时，则可以同样地处理，显示Detached字样。传感器的状态检测也是要常态化的。
